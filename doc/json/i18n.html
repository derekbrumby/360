i18n

ForgeJS supports internationalization of everything, from text to media. This section allows to configure the internationalization, by specifying each available locale. It is recommended to follow a simple scheme to identify each locale, such as en-US or fr-FR.

Example :

{
  "enabled": false,
  "default": "en-US",
  "auto": false,
  "locales": [
    {
      "uid": "en-US",
      "name": "English",
      "files": [
        {
          "key": "project.en-US",
          "url": "./i18n/en-US/locale.json"
        }
      ]
    }
  ]
}

i18n #

name type prop description
enabled boolean default: true
 Is the i18n support enabled for this project ?

Example : 
false
default string The default locale UID to use.

Example : 
"en-US"
auto boolean Is the device default browser locale used if available ?

Example : 
false
locales array<object> The list of each locale configuration. Each member is an instance of an i18n locale.

i18n-locale #

name type prop description
uid string required
 The UID of the locale, we recommend a code respecting the RFC 5646 for better identification.

Example : 
"en-US"
name string required
 The name of the locale.

Example : 
"English"
files array<object> required
 The list of files containing the locale. Each member is an instance of a i18n locale file.

i18n-locale-files #

name type prop description
key string required
 It is the unique identifier for this file.

Example : 
"project.en-US"
url string required
 The file can be local or on a distant server, but must be a valid JSON one.

Example : 
"./i18n/en-US/locale.json"
