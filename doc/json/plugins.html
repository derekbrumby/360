Plugins

A plugin can add elements and functionnalities that are not available through ForgeJS. It can be as simple as a button reacting to a click by going to the next scene, to a complex plugin of chromakey, detouring in real time something on top of a green screen.

Example :

{
  "prefix": "./plugins",
  "engines": [
    {
      "uid": "com.awesome.plugin",
      "prefix": "./other-plugin-folder/",
      "url": "AwesomePlugin",
      "manifest": "manifest.json"
    }
  ],
  "instances": [
    {
      "uid": "awesome-plugin-instance",
      "engine": "com.awesome.plugin",
      "tags": [
        "plugin",
        "beach",
        "awesome"
      ],
      "scenes": [
        "scene-0",
        "scene-2"
      ],
      "options": {
        "awesomeness": 360,
        "key": "i18n-awesome",
        "another_options": {
          "target": "scene-1",
          "enabled": true
        }
      },
      "data": {},
      "actions": [
        "action-awesome-thing",
        "action-more-awesome"
      ],
      "events": {
        "onClick": [
          "action-awesome-thing"
        ]
      },
      "index": 10,
      "enabled": true,
      "keep": false
    }
  ],
  "configurations": [
    {
      "data": {},
      "options": {
        "key": "overwritting-previous-key",
        "missing-options": 42
      },
      "actions": [
        "action-awesome-thing",
        "action-more-awesome"
      ],
      "events": {
        "onDrag": [
          "action-more-awesome"
        ]
      }
    }
  ]
}

plugin #

name type prop description
engines array<object> required
 List of plugin engines. Each member is an instance of a plugin engine.
instances array<object> required
 List of plugin instances. Each member is an instance of a plugin instance.
configurations array<object> required
 List of plugin configurations. Each member is an instance of a plugin configuration.
prefix string The prefix to put before the path of each plugin. Can be override by the prefix properties in the plugin engine declaration.

Example : 
"./plugins"

plugin-engine #

name type prop description
uid string required
 The UID of the engine.

Example : 
"com.awesome.plugin"
prefix string The prefix of the url.

Example : 
"./other-plugin-folder/"
url string The url/root of the Plugin relative to the prefix.

Example : 
"AwesomePlugin"
manifest string The manifest to read to know the configuration of the plugin.

Example : 
"manifest.json"

plugin-instance #

name type prop description
uid string required
 The UID of the instance.

Example : 
"awesome-plugin-instance"
tags array<string> required
 A list of tags.
options object required
 The options of this instance, specific and different for each plugin.

Example : 
{
  "awesomeness": 360,
  "key": "i18n-awesome",
  "another_options": {
    "target": "scene-1",
    "enabled": true
  }
}
actions array<string> required
 A list actions associated to this plugin.
events object required
 Events to react to, specific to each plugin.

Example : 
{
  "onClick": [
    "action-awesome-thing"
  ]
}
engine string The UID of the engine the instance is of.

Example : 
"com.awesome.plugin"
scenes array<string> The list of UID of scenes associated to the instance.
data object The eventual data associated to the plugin, specific to each plugin.

Example : 
{}
index number The index of the plugin on the screen.

Example : 
10
enabled boolean Is the instance enabled ?

Example : 
true
keep boolean Do we need to keep the plugin ?

Example : 
false

plugin-configuration #

name type prop description
data object The eventual data associated to the plugin, specific to each plugin.

Example : 
{}
options object The options of this instance, specific and different for each plugin.

Example : 
{
  "key": "overwritting-previous-key",
  "missing-options": 42
}
actions array<string> A list of actions associated to this plugin.
events object Events to react to, specific to each plugin.

Example : 
{
  "onDrag": [
    "action-more-awesome"
  ]
}
