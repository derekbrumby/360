Scene

A scene is essentially composed of a 360º media to display, but also local configuration to itself. It can contain a specific view or camera configuration for example. This is also where hotspots are defined, along with the director's cut and specific plugins configuration.

scene #

name type prop description
uid string The UID of the scene.

Example : 
"scene-0"
name string The name of the scene or its i18n key.

Example : 
"My Scene"
slug string The slug name of the scene or its i18n key.

Example : 
"my-scene"
description string The description of the scene or its i18n key.

Example : 
"This is a super scene with awesome things in it"
background string The background color of the scene.

Example : 
"#000"
fx string The UID of the special effect (FX) set to apply to the scene.

Example : 
"fx-0"
tags array<string> The list of tags associated to the scene.
sync array<string> The list of scene uids to sync this scene with.
thumbnails array<object> The configuration of a thumbnail associated to a scene. Each member is an instance of a scene thumbnail.
media object The media object associated to the scene. An instance of a scene media.
sound object An instance of a sound.
playlists object An instance of the global playlist configuration.
view object An instance of the view configuration.
camera object An instance of the camera object.
audio object An instance of the audio configuration.
hotspots array<object> The list of hotspots associated to this scene. Each member is and instance of a hotspot.
director object An instance of a director configuration.
plugins object An instance of the plugin object.
events object An instance of the scene events configuration.

name type prop description

Example : 
"thumb-0"

    scene-media-preview #

    name type prop description
    url string The URL for the preview of the media, with a {face} parameter to specify the face when parsing. Only one image per face is authorized, so no other parameter can be specified.

    Example : 
    "multi/preview/{face}.jpg"
    tile number The size of a tile, which corresponds to the width and the height of a tile. It will be used with the width and height to compute the value of the {x} and {y} patterns.

    Example : 
    512

    scene-media-source #

    name type prop description
    format string The format of the media: equi is for an equirectangular resource (the most common one) and cube is for a cubemap resource (less common but more efficient to display). Can be :
    • equi
    • cube
    • flat
    • hotspot


    Example : 
    "equi"
    streaming string The streaming method: either a html5 video (usually .mp4 or .webm), or a dash flux (.mpd). The authorized values are :
    • html5
    • dash


    Example : 
    "html5"
    url string This property is only set when the media is an image. The file can be local or on a distant server, but must be a valid image format.

    Example : 
    "./images/scene-0.jpg"
    fov object The field of view of a media object. An instance of a scene media fov object.
    pattern string The pattern to respect in case of a multi tiles media. Four parameters are authorized:
    • {face} is for the face, in case of a cubemap media
    • {level} is for the level of quality of the tile, in case of a multi resolution media
    • {x} if for the x coordinates of the tile
    • {y} if for the y coordinates of the tile


      • Example : 
      "multi/equi/{level}/{y}/{x}.jpg"
    levels array<object> This property is only set when the media is a video. The files can be local or on a distant server, but must be a valid video format. If the array is composed of multiple URLs, it is considered that there are multiple qualities to consider. Each member is an instance of a scene source level.
    order string default: RLUDBF
    		 Order of the layout in the cubemap media, ie where are the faces in the media, starting from the top left of the media. Needs to be a 6-char string, constitued with all of the following letter:
    • R for Right
    • L for Left
    • U for Up
    • D for Down
    • B for Back
    • F for Front


    Example : 
    "RLUDBF"
    tile number The size of a tile, in case the chosen format is cube. The size corresponds to the width and the height of a face.

    Example : 
    512
    faces array<object>

    scene-media-level #

    name type prop description
    name string If set, specifies the {level} pattern for this level. If not, the value will be the index of the level in the levels array.

    Example : 
    "4K"
    url string If set, overrides any computed URL by this one. If not, the URL will be computed from other values

    Example : 
    "multi/video-2K.mp4"
    device object The device requirements.

    Example : 
    {
  "safari": false,
  "webvr": true,
  "os": "Windows"
}
    pattern string The pattern to respect in case of a multi tiles media. Four parameters are authorized:
    • {face} is for the face, in case of a cubemap media
    • {level} is for the level of quality of the tile, in case of a multi resolution media
    • {x} if for the x coordinates of the tile
    • {y} if for the y coordinates of the tile


      • Example : 
      "multi/equi/{level}/{y}/{x}.jpg"
    width number The width of the source. It will be used with the height and tile to compute the value of the {x} and {y} patterns.

    Example : 
    2048
    height number The height of the source. It will be used with the width and tile to compute the value of the {x} and {y} patterns.

    Example : 
    1024
    tile number The size of a tile, which corresponds to the width and the height of a tile. It will be used with the width and height to compute the value of the {x} and {y} patterns.

    Example : 
    512

    scene-media-options #

    name type prop description
    autoPlay boolean Does the video need to be played automatically at start ?

    Example : 
    false
    loop boolean Does the video need to be looped automatically at the end of playing ?

    Example : 
    false
    volume number default: 1
    max: 1
    		 The volume of the media.

    Example : 
    1
    startTime number Set the current position (in seconds) of the video playback.

    Example : 
    0
    autoPause boolean default: true
    		 Does the video need to be paused automatically when the page is not visible anymore ?

    Example : 
    true
    autoResume boolean Does the video need to be resumed automatically when the page is visible again ?

    Example : 
    false
